Начальная дата и время наблюдений устанавливается при конструировании объектов и в процессе работы не меняется.
Класс должен содержать необходимые служебные методы (конструкторы, деструктор, …).
Класс должен предоставлять следующие операции:
8) сохранить историю наблюдений в файл и считать историю наблюдений из файла.
1) узнать начальные дату и время наблюдений,
2) добавить наблюдение,
3) узнать температуру в наблюдении, выбранном по дате и времени,
4) выдать серию наблюдений для выбранной даты,
5) найти среднюю температуру для выбранной даты, выбранного месяца, за всю историю наблюдений,
7) найти среднюю дневную или ночную температуру для выбранного месяца,
Программа должна иметь простое меню для выбора необходимых операций.
Eng: (Google Translator)
Made a Thermometer class.
The class must keep a history of temperature observations for one calendar (not leap) year. Observations are described by date (day, month, year) and time (in hours). When a new observation arrives for an existing date and time, the old observation is replaced. The temperature is set in degrees Celsius.
The initial date and time of observations is established during the construction of objects and does not change during the work.
The class must contain the necessary utility methods (constructors, destructor, ...).
The class must provide the following operations:
8) save the history of observations in a file and read the history of observations from the file.
1) find out the initial date and time of observations,
2) add observation,
3) find out the temperature in the observation selected by date and time,
4) issue a series of observations for the selected date,
5) find the average temperature for the selected date, the selected month, for the entire history of observations,
7) find the average day or night temperature for the selected month,
The program should have a simple menu to select the necessary operations.
